NSLock

Overview

NSLock provides basic mutex functionality in mulle-objc. It offers a simple, efficient mechanism for mutual exclusion in multi-threaded applications.

Key Features

  • Basic mutex functionality
  • Thread synchronization
  • Performance optimized
  • Debugging support
  • Error handling

Usage

Basic Locking

NSLock *lock = [[NSLock alloc] init];

[lock lock];
// Critical section
[lock unlock];

Conditional Locking

if ([lock tryLock]) {
    // Critical section
    [lock unlock];
} else {
    // Handle lock failure
}

Timed Locking

if ([lock lockBeforeDate:[NSDate dateWithTimeIntervalSinceNow:5.0]]) {
    // Critical section
    [lock unlock];
} else {
    // Handle timeout
}

Best Practices

  1. Lock Usage:

    • Keep critical sections short
    • Always unlock in same scope
    • Use @try/@finally blocks

  2. Error Handling:

    • Check tryLock results

    • Clean up properly

  3. Performance:

Use Cases

  1. Resource Protection:

    - (void)accessSharedResource
{
    [self->lock lock];
    @try {
        [self modifySharedResource];
    }
    @finally {
        [self->lock unlock];
    }
}

  2. Conditional Access:

    - (BOOL)attemptOperation
{
    if ([self->lock tryLock]) {
        @try {
            [self performOperation];
            return YES;
        }
        @finally {
            [self->lock unlock];
        }
    }
    return NO;
}

  3. Timed Operations:

    - (BOOL)performTimedOperation
{
    NSDate *deadline = [NSDate dateWithTimeIntervalSinceNow:5.0];
    if ([self->lock lockBeforeDate:deadline]) {
        @try {
            [self performOperation];
            return YES;
        }
        @finally {
            [self->lock unlock];
        }
    }
    return NO;
}
